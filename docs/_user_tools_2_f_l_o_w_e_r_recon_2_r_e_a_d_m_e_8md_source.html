1 # FLOWERRecon
2 
3 FLOWERRecon will take a reconstructed vertex (and hits from the associated trigger) from the DataModel and run the FLOWER low-energy reconstruction algorithm
4 
5 ## Data
6 
7 * Performs setup of FLOWER reading in from the DataModel
8  * Geometry from `WCSimGeomTree`
9  * PMT dark rate from `IDPMTDarkRate`
10  * Number of PMTs from `IDNPMTs`
11 * For each reconstructed vertex that pass a filter (see `ReconFilter`)
12  * Read in the vertex from the `ReconInfo`
13  * Get the hits in the trigger from `IDWCSimEvent_Triggered`
14  * Run the FLOWER external program to get the reconstructed energy
15

17 ## Configuration
18 
19 ### Parameters that specify which reconstructed events to use
20 ```
21 input_filter_name NAME
22 nhitsmin NUM
23 nhitsmax NUM
24 ```
25 
26 * `input_filter_name` Name of the `ReconInfo` object to take from the map `RecoInfoMap`
27  * If ALL takes the all reconstructed events from the global object `RecoInfo`
28 * `nhitsmin` When the number of hits in the trigger is below this number, FLOWER will not be called
29 * `nhitsmax` When the number of hits in the trigger is above this number, FLOWER will not be called
30 
31 ### FLOWER running parameters
32 ```
33 n_working_pmts NUM
34 detector_name NAME
35 overwrite_nearest_neighbours BOOL
36 flower_verbose NUM
37 ```
38 * `n_working_pmts` Use this to mask PMTs. If `n_working_pmts` more than `number of PMTs in simulated geometry`, this is set to `number of PMTs in simulated geometry`. Default is `number of PMTs in simulated geometry`
39 * `detector_name` Used by FLOWER to set default values of e.g. nearest neighbour distance
40 * `overwrite_nearest_neighbours` `false`: read in the nearest neighbour file from `$FLOWERDIR/data/`. `true`: overwrite this file with a recalculation. Takes ~5 minutes to calculate for ~40k PMTs
41 * `flower_verbose` Verbosity level for FLOWER internally. Runs from 0 (low verbosity) to 9 (high verbosity)
42

47 * `verbose` Verbosity level. Runs from 0 (low verbosity) to 9 (high verbosity)
